# Conversions-Checkliste

Ziel dieser Checkliste ist **Vertrauen durch Ruhe und Klarheit** – ausdrücklich nicht durch Verkaufsdruck, künstliche Verknappung oder Dropshipping-typische Taktiken.

## Produktseite

* [ ] Mehrere hochwertige Produktbilder aus unterschiedlichen Blickwinkeln
* [ ] Produktbeschreibung folgt der [Vorlage](product-description-template.md) (80–150 Wörter, ruhiger Ton)
* [ ] Materialangaben vorhanden und korrekt (keine erfundenen Details)
* [ ] Klarer, unaufdringlicher Call-to-Action
* [ ] Keine künstliche Verknappung ("nur noch 2 auf Lager" nur, wenn faktisch zutreffend)
* [ ] Keine erfundenen oder unbelegten Kundenbewertungen

## Vertrauenselemente

* [ ] Sichtbarer Zugang zu Kontaktinformationen
* [ ] Sichtbarer Zugang zu Versand- und Rückgabeinformationen (sobald final, siehe [Rechtliche Platzhalter](legal-placeholders.md))
* [ ] Über-uns-Bereich mit echter Markengeschichte statt generischem Text
* [ ] Konsistente, ruhige Bildsprache über die gesamte Seite hinweg

## Sprache & Ton

* [ ] Kein aggressiver Verkaufston, keine Ausrufezeichen-Häufung
* [ ] Keine Floskeln wie „zeitlos elegant" oder „ein Must-have" (siehe [Markenidentität](brand-identity.md))
* [ ] Keine unbelegten Superlative ("Bestseller", "meistgeliebt") ohne Datenbasis

## Technisch

* [ ] Ladezeit der Produktseiten geprüft
* [ ] Mobile Darstellung geprüft
* [ ] Checkout-Prozess getestet (keine Reibung, klare Schritte)

{% hint style="info" %}
Diese Liste ist bewusst konservativ gehalten. Sie priorisiert langfristiges Vertrauen gegenüber kurzfristiger Konversionsoptimierung durch Druckmittel.
{% endhint %}

## Site-weite Prüfung

Ergänzend zur Produktseiten-Checkliste oben — einmal komplett durchgehen, bevor der Shop live geht oder größere Traffic-Kampagnen starten:

| Bereich | Prüfpunkt |
|---|---|
| Header | Logo klickbar zur Startseite, Suchfunktion sichtbar, Warenkorb-Icon mit Live-Anzahl |
| Navigation | Alle Kategorien erreichbar, keine toten Links, konsistente Reihenfolge (siehe [Website-Struktur](website-structure.md)) |
| Mobile Ansicht | Navigation per Hamburger-Menü nutzbar, Schriftgrößen lesbar, keine horizontal scrollenden Elemente |
| Produktbilder | Einheitlicher Bildstil (Licht, Hintergrund), mind. 2–3 Ansichten je Produkt, Zoom-Funktion |
| Produkttexte | Folgen [Produktbeschreibung-Vorlage](product-description-template.md), keine Platzhalter-Reste live |
| Warenkorb | Mengenänderung funktioniert, Preis inkl. MwSt. sichtbar, klarer Weg zum Checkout |
| Checkout-Vertrauen | Zahlungsmethoden sichtbar vor Checkout-Start, SSL/Sicherheitshinweis, keine überraschenden Zusatzkosten |
| Footer | Alle rechtlichen Links vorhanden und funktionieren (siehe unten) |
| Rechtliche Links | Impressum, Datenschutz, AGB, Widerruf, Versand & Zahlung, Kontakt — alle erreichbar, keine 404 (siehe [Rechtliche Platzhalter](legal-placeholders.md)) |
| Ladegeschwindigkeit | Produktbilder komprimiert, keine unnötig großen Apps/Skripte im Theme |
| Suchfunktion | Liefert relevante Ergebnisse für Produktnamen und Kategorien, keine Leerergebnisse bei naheliegenden Suchbegriffen |
| Kategorien | Jede Kategorie hat Inhalt (siehe [Kategorie-Seiten](category-pages.md)), keine leeren Kollektionsseiten |
| Leere Seiten | Keine Platzhalter-Seiten ("Coming soon", leere Blog-Kategorie) öffentlich verlinkt |
| 404-Fehler | Eigene, ruhige 404-Seite mit Link zurück zur Startseite/Kategorien statt Standard-Shopify-Fehlerseite |
| Fehlende Trust-Elemente | Trust-Bereich auf Startseite vorhanden (siehe [Homepage](homepage.md)), Trust-Badges auf Produktseiten |
| Unklare Versandinfos | Versandzeit und -kosten vor dem Checkout ersichtlich, nicht erst in den AGB versteckt |
| Fehlende Kontaktinfos | E-Mail/Kontaktformular von jeder Seite aus max. 2 Klicks erreichbar |

## Prioritätenliste

**1. Was muss sofort geändert werden?**
* Alle Platzhalter in [legal-placeholders.md](legal-placeholders.md) durch echte, geprüfte Angaben ersetzen (rechtliches Risiko ohne gültiges Impressum/Widerrufsrecht)
* Eigene 404-Seite einrichten, falls noch Standard-Shopify-Fehlerseite aktiv
* Keine leeren oder "Coming soon"-Seiten öffentlich verlinkt lassen

**2. Was bringt am schnellsten Umsatz?**
* Trust-Bereich und klarer Hero-CTA auf der Startseite (siehe [Homepage](homepage.md))
* Vollständige Produktseiten für alle aktiven Produkte (Materialhinweise, FAQ, Trust-Badges — siehe [Produktseiten-Vorlage](product-page-template.md))
* Mobile Sticky Add-to-Cart aktivieren

**3. Was ist wichtig für Vertrauen (mittelfristig)?**
* Versand- und Rückgabeinformationen klar und konsistent an allen Touchpoints (Produktseite, Footer, Checkout)
* Über-uns-Bereich mit echter Markengeschichte statt Platzhaltertext
* Konsistente Bildsprache über den gesamten Shop

**4. Was kann später gemacht werden?**
* Blog/Journal-Bereich (siehe [SEO-Plan](seo-plan.md))
* Erweiterte Bestseller-Kategorie, sobald echte Verkaufsdaten vorliegen
* Feinschliff an interner Verlinkung zwischen Kategorien
